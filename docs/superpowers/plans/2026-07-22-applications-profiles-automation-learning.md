# Applications, profiles, automation, and learning

1. Add process-only app rules with category, profile, intensity, fullscreen protection, priority, and deterministic conflict resolution.
2. Keep built-in profiles immutable; allow duplicate/edit/delete for custom profiles and safe previews.
3. Add bounded deterministic automation for time, app, fullscreen, luminance, session, display, and power context.
4. Explain every decision using result, primary reason, confidence, source, and responsible rule or learned preference.
5. Store aggregate learning only; add forget-app, reset-all, disable, and export controls.
6. Limit content sampling to useful categories, active display/window, low resolution, low frequency, and memory-only processing.
7. Gate: low-confidence learning stays weak; fullscreen game/video/presentation protection cannot be overridden automatically.
